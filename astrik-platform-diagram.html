🏗️ Arquitectura del Sistema
Vista general de componentes y su comunicación. Todo corre 100% local con llama.cpp para inferencia.
OpenCode
CLI / IDE principal. Desde acá lanzamos sesiones y sub-agentes para construir cada módulo. Es el punto de entrada del desarrollador.
Orchestrator Core
El cerebro del sistema. Recibe objetivos, los descompone, asigna agentes y trackea el estado de cada tarea. Usa FastAPI + Redis + PostgreSQL + NATS.
Sistema de Agentes
Agentes especializados (planner, backend, qa, devops, architect, security). Cada uno con un rol, herramientas y modelo asignado.
Memoria Persistente
Tres capas: Redis (short-term), Qdrant (memoria vectorial/semántica), PostgreSQL (persistencia estructurada). RAG + Git indexing incluidos.
Event Bus (NATS)
Comunicación asíncrona entre agentes. Eventos de tareas, workflows, errores. Permite desacoplamiento total entre componentes.
llama.cpp Runtime
Motor de inferencia directa sobre GPUs. Soporta modelos GGUF: Hermes 3, DeepSeek-Coder, Qwen2.5-Coder, Phi-4. Control total de VRAM y scheduling.

🤖 Sistema de Agentes
Cada agente tiene un rol específico, herramientas asignadas y un modelo de IA optimizado para su función.
Planner Agent
Rol: Orquestación y coordinación
Modelo: Hermes 3 (llama.cpp)
Funciones: Recibe objetivos, divide en tareas, genera DAGs de dependencias, prioriza, asigna agentes
Eventos: Publica TASK_CREATED, WORKFLOW_STARTED
Backend Agent
Rol: Generación de APIs y lógica backend
Modelo: DeepSeek-Coder (llama.cpp)
Funciones: FastAPI, schemas Pydantic, modelos SQLAlchemy, tests, async
Eventos: Publica TASK_COMPLETED, consume TASK_ASSIGNED
QA Agent
Rol: Validación y calidad de código
Modelo: Qwen2.5-Coder (llama.cpp)
Funciones: Unit tests, integration tests, linting, static analysis, security scan
Eventos: Publica TASK_FAILED/TASK_PASSED
DevOps Agent
Rol: Infraestructura y deploy
Modelo: Phi-4 (llama.cpp)
Funciones: Dockerfiles, docker-compose, deploy local, CI/CD, monitoring
Eventos: Publica DEPLOY_STARTED, DEPLOY_FINISHED
Architect Agent
Rol: Diseño y documentación arquitectural
Modelo: Hermes 3
Funciones: Diagramas, contratos API, decisiones técnicas, documentación
Herramientas: filesystem, docs
Security Agent
Rol: Auditoría de seguridad
Modelo: Qwen2.5-Coder
Funciones: Análisis de vulnerabilidades, secretos, validación de dependencias
Herramientas: static analysis, bandit

🔄 Workflow Autónomo
Cómo fluye un objetivo desde que el usuario lo pide hasta que se despliega automáticamente.
Flujo Completo
  1. Usuario escribe un objetivo en lenguaje natural
  2. Planner Agent recibe el objetivo y lo descompone en tareas (DAG)
  3. Cada tarea se asigna al agente especializado correspondiente
  4. Agentes trabajan en paralelo, comparten contexto vía Memory Layer
  5. QA Agent valida cada entregable automáticamente
  6. DevOps Agent dockeriza y despliega localmente
  7. Dashboard muestra el estado en tiempo real
Modelo de Eventos
TASK_CREATED → Tarea nueva en el sistema
TASK_ASSIGNED → Asignada a un agente
TASK_RUNNING → Agente trabajando
TASK_COMPLETED → Tarea terminada
TASK_FAILED → Error, se reintenta
WORKFLOW_FINISHED → Flujo completo

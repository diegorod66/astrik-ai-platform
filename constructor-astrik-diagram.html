Subproyecto de Agentes Autonomos con LangGraph v1.0 Diseno
[+] Que es Constructor Astrik?
Es el subproyecto que construye la infraestructura de agentes autonomos. No es el proyecto final (Astrik AI Platform), es la fabrica que va a construir ese proyecto final de forma automatica.
Objetivo
Construir agentes que funcionen por si solos, escuchen eventos, tomen decisiones y ejecuten tareas sin intervencion manual.

Problema actual: Los agentes son scripts CLI.
Solucion: Orchestrator con LangGraph + Event Bus NATS.
Arquitectura Target
LangGraph Workflow engine + state machine
FastAPI API REST
NATS Event bus entre agentes
PostgreSQL Persistencia de estado
Redis Colas + cache
Qdrant Memoria compartida

[*] Orchestrator con LangGraph
LangGraph reemplaza el diseno manual de state machine, scheduler y workflow engine. Todo se vuelve un grafo de nodos (agentes) conectados por aristas (eventos).

[!] Hecho a Medida vs LangGraph
Comparacion directa entre el diseno original (FastAPI + state machine manual) y LangGraph.
ComponenteDiseno Original (FastAPI puro)LangGraph
State Machine Implementar manual con clases + enum + transiciones Built-in: StateGraph con nodos y aristas. Checkpointing automatico.
Workflow DAG Router manual con if/else + cola Redis Natural: aristas condicionales entre nodos. Routing implicito.
Persistencia SQLAlchemy + tablas task/workflow/agent manuales Checkpointer: guarda/restaura estado automaticamente en PostgreSQL
Agentes Router manual que invoca agentes por CLI AgentNode: wrapper LangGraph que usa modelos + tools nativamente
Tools Funciones sueltas sin registro central Tool registry: tools decoradas, disponibles para cualquier nodo
Streaming Implementar con WebSockets + asyncio.Queue Nativo: streaming de eventos del grafo en tiempo real
Retry / Error Mecanismo manual con contadores + colas Built-in: nodos pueden tener aristas de error + limite de reintentos
Human-in-loop Implementar con endpoints + webhooks Nativo: interrupt() pausea el grafo, espera input humano
Codigo estimado ~2000 lineas (solo scaffolding) ~300 lineas (logica de dominio pura)
Mantenimiento Alto: cada cambio en workflow = cambiar state machine Bajo: agregar/quitar nodos = agregar/quitar edges
Veredicto
LangGraph es superior para este proyecto porque:
  • Elimina ~80% del codigo de infraestructura
  • Ya resuelve state machine, persistencia, streaming, retry
  • Es el estandar de facto para sistemas multi-agente
  • Nos deja enfocar en logica de dominio
Riesgo
LangGraph depende de LangChain, que es pesado (~200MB con dependencias). Si en el futuro LangChain cambia, LangGraph puede romperse.

Mitigacion: Nuestra logica de negocio vive en tools/ y agents/, no en LangGraph. Si cambia, solo refactorizar graph.py.

Orchestrator Agent
Rol: Nodo central que ejecuta el grafo LangGraph
Modelo: Hermes 3
Funcion: Recibe objetivos, ejecuta el grafo, orquesta nodos, publica eventos
Eventos: Consume OBJECTIVE_SUBMITTED, publica WORKFLOW_COMPLETED
Version Agent (nuevo)
Rol: Agente de versionado continuo
Modelo: Phi-4 (tasks rapidas)
Funcion: Snapshot automatico al completar cada hito, bump de version, CHANGELOG
Eventos: Consume WORKFLOW_STAGE_COMPLETED, publica VERSION_BUMPED
